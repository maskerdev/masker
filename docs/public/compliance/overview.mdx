## What Masker is

Masker is a **privacy layer** that sits between your voice agent and the LLM. It is not, on its own, a HIPAA-compliant service. Compliance is a property of your overall system. Masker exists to make that system substantially easier to certify.

## What Masker does for compliance

### Reduces what crosses a trust boundary

By the time a payload leaves your VPC and hits OpenAI (or any other model), every detected identifier has been replaced with a token. This shrinks the set of services in your HIPAA boundary — OpenAI no longer processes PHI on your behalf, because they don't see PHI.

### Produces auditable evidence

Every redaction event is logged to an HMAC-chained, append-only audit log. Reports can be generated on demand (per session, per agent, per account). Auditors get the same artifact your security team gets.

### Stays inside your VPC (when you choose)

Masker can run as a Docker container in your AWS / GCP / Azure account. PHI never leaves your network on the way to the model — it leaves as tokens. This is the deployment we recommend for any production healthcare workload.

### Doesn't store PHI by default

The default configuration:

- Does **not** log raw payloads
- Does **not** persist transcripts
- Does **not** retain audio
- Stores only tokens, events, and metadata

You can opt **in** to encrypted payload retention per agent, with a configurable window. Default is off.

## What Masker doesn't do

Equally important to be clear about:

- **Masker is not a substitute for a security program.** You still need access control, vendor management, breach response, training, and the rest of HIPAA's administrative safeguards.
- **Masker doesn't sign your BAA with OpenAI / Anthropic / etc.** You should not need one in most architectures (because they don't see PHI), but you decide that with your counsel.
- **Masker is not certified.** SOC 2 is in flight. HITRUST is not. If a customer requires either before a pilot, we can't yet meet that bar — the answer is to wait for SOC 2 Type 1.

## How we got here

A few choices that shape the compliance posture:

- **Rust + minimal dependencies.** Smaller supply chain, fewer CVEs, easier to audit.
- **No telemetry.** The container does not phone home. There is no analytics call you'd need to disclose.
- **No SaaS-only mode.** Customer VPC has been a first-class deployment target since v0. We didn't bolt it on later.
- **Open source.** The detection, tokenization, and audit code is in the public repo. Auditors don't have to take our word for any of it.
