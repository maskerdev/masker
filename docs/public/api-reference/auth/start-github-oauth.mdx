Begins the GitHub OAuth login flow. This endpoint is **not** an API call you make from your code — it's a redirect target you point a browser at.

## Endpoint

```
GET /api/v1/auth/github
```

## Behavior

1. Generates a one-time CSRF token and stores it in a short-lived cookie (`masker_oauth_state`).
2. Redirects to `https://github.com/login/oauth/authorize?...` with Masker's GitHub OAuth app `client_id` and the configured callback URL.
3. The user authenticates with GitHub and approves access.
4. GitHub redirects back to [`/api/v1/auth/github/callback`](/api-reference/auth/oauth-callback) with `code` and `state` query parameters.

## Query parameters

| Param | Type | Required | Description |
| --- | --- | --- | --- |
| `redirect_to` | string | no | Path to redirect to after successful login. Defaults to `/portal`. Must be a relative path on the same origin. |

## Example

```html
<a href="https://masker-voice.fly.dev/api/v1/auth/github?redirect_to=/portal/agents">
  Sign in with GitHub
</a>
```

## Response

Always a `302` redirect to GitHub. No JSON body.

## Security notes

- The CSRF token in `masker_oauth_state` must round-trip via the callback. Mismatches reject the login.
- `redirect_to` is validated to be a same-origin relative path. External URLs are rejected.
- The OAuth app requests scope `read:user` only. Masker does not request repo, org, or write scopes.
