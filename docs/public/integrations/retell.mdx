**Bottom line up front:** Retell AI supports custom LLM endpoints. Point your Retell agent's LLM URL at a Masker proxy URL and every chat completion is routed through Masker, which tokenizes PHI before forwarding upstream and rehydrates the original values in the response. The integration pattern is identical to [Vapi](/integrations/vapi) — the only difference is where you paste the URL inside Retell's dashboard.

<Note>
**Status:** Retell platform integration is in beta as of May 2026. The portal supports Retell agents under the same OAuth flow as Vapi. To request beta access, see [Beta access](/get-started/beta-access).
</Note>

## Why route Retell through Masker

Retell handles the phone connection, low‑latency STT, and TTS, and lets you bring your own LLM via a custom endpoint. That custom endpoint is the perfect place to insert HIPAA controls: by replacing the LLM URL with a Masker proxy URL, you get tokenization, vault, rehydration, and audit logging without changing your agent's behavior or your prompt.

## Architecture

```
Caller ↔ Retell ──► Masker proxy ──► OpenAI ──► Masker ──► Retell ↔ Caller
                    │                            │
                    detect + tokenize             rehydrate
```

Per‑agent proxy URL shape:

```
https://masker-voice.fly.dev/proxy/{agent_id}/v1/chat/completions
```

## Step 1 — Create a Retell agent in the Masker portal

<Steps>
  <Step title="Sign in">
    Open [masker-voice.fly.dev/portal/login](https://masker-voice.fly.dev/portal/login) and click **Sign in with GitHub**.
  </Step>
  <Step title="Click 'New Agent' → choose 'Retell'">
    The wizard creates an agent record and provisions a per‑agent proxy URL. (Today this URL is identical in shape to the Vapi one — the wizard just records that the agent is wired to Retell so the audit reports show the right BAA chain.)
  </Step>
  <Step title="Copy the proxy URL">
    `https://masker-voice.fly.dev/proxy/{agent_id}/v1/chat/completions`
  </Step>
</Steps>

## Step 2 — Configure Retell to use Masker

In the [Retell dashboard](https://app.retellai.com):

1. Open the agent you want to make HIPAA‑safer.
2. In **LLM settings**, switch the LLM type to **Custom LLM**.
3. Paste the Masker proxy URL into **Custom LLM URL**.
4. Set **Model** to `gpt-4o-mini` (or any OpenAI model your account has access to — Masker forwards the model field through).
5. Save.

```yaml
# Retell agent config (example)
llm_type: custom
custom_llm_url: https://masker-voice.fly.dev/proxy/agt_01HYZ.../v1/chat/completions
model: gpt-4o-mini
temperature: 0.4
```

## Step 3 — Place a test call

Use Retell's **Test agent** widget. Try:

> "My name is Marcus Lee, my insurance ID is AET-882144, and I'm calling about my prescription refill."

Open the **Sessions** tab in the Masker portal. The session record shows the raw transcript on one side and the masked transcript on the other:

```
Raw:    "My name is Marcus Lee, my insurance ID is AET-882144..."
Masked: "My name is MSKV1.person_name.K_HEALTHCARE.d4e1...,
         my insurance ID is MSKV1.account_number.K_HEALTHCARE.f2a7..."
```

## Step 4 — Verify in your LLM provider's logs

Same verification step as Vapi — open [platform.openai.com/logs](https://platform.openai.com/logs) and confirm only Masker tokens appear in the chat completion input.

## What's the same as Vapi, what's different

| | Vapi | Retell |
| --- | --- | --- |
| Integration mechanism | Custom LLM URL | Custom LLM URL |
| Per‑agent proxy URL | ✅ same shape | ✅ same shape |
| Webhook for assistant‑request events | ✅ supported | ⏳ Retell uses inline config, no webhook needed |
| Tool calls forwarded with masking | ✅ | ✅ |
| BAA chain in audit report | Vapi → OpenAI → Deepgram → ElevenLabs | Retell → OpenAI → (Retell's STT/TTS chain) |
| Latency overhead | 45–95 ms | 45–95 ms |
| Production readiness | ✅ live | 🚧 beta |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Retell shows 'LLM call failed: 502'">
    Same as the Vapi case — Masker received the request but the upstream LLM call failed. Check OpenAI key validity, model availability, and Masker `/health`. Always include the `request_id` from the error envelope when filing.
  </Accordion>
  <Accordion title="Caller hears tokens like 'MSKV1...'">
    Rehydration failed. Open the session in the Masker portal and check **Detection details** for `rehydrated: false` flags. The most common cause is the LLM truncating a token mid‑string.
  </Accordion>
  <Accordion title="Retell tool calls are leaking PHI">
    Confirm the tool call is going through the Masker proxy (not directly to OpenAI). Some teams accidentally configure tools to bypass the LLM URL. See [Tokenization](/masking/tokenization) for tool‑call masking details.
  </Accordion>
</AccordionGroup>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Does Retell AI support HIPAA?">
    Retell offers a BAA on its enterprise tier. Masker is one node in the HIPAA chain — adding Masker reduces the surface area where PHI lives in the LLM hop, but you still need BAAs with Retell, your STT/TTS providers, and the LLM provider.
  </Accordion>
  <Accordion title="Can I use Masker with Retell's built-in LLM?">
    Not today — Masker only routes through the **Custom LLM** path. If you want PHI redaction in the LLM hop, switch to Custom LLM and point it at Masker.
  </Accordion>
  <Accordion title="What's the latency overhead?">
    45–95 ms in steady state, identical to the Vapi pattern.
  </Accordion>
  <Accordion title="When will Retell move from beta to GA?">
    Targeting the May 30, 2026 production launch.
  </Accordion>
</AccordionGroup>
