---
title: "Logout"
api: "POST /api/v1/auth/logout"
description: "End the current session"
---

Logs out the current user by invalidating the session and clearing the cookie.

## Endpoint

```
POST /api/v1/auth/logout
```

## Authentication

Requires the `masker_session` cookie.

## Request

No body required.

```bash
curl -X POST -H "Cookie: masker_session=ey..." \
  https://masker-voice.fly.dev/api/v1/auth/logout
```

## Response

```
HTTP/1.1 204 No Content
Set-Cookie: masker_session=; Max-Age=0; Path=/; HttpOnly; Secure
```

The session row is deleted server-side and the cookie is cleared. The cookie value is **permanently** invalid — you cannot resurrect it by re-sending. To get a new session, restart the OAuth flow.

## Errors

| Status | Code | Description |
| --- | --- | --- |
| `401` | `unauthenticated` | No session cookie or already invalid |

Calling logout on an already-invalid session is **not** an error — Masker returns `204` regardless. This makes the call idempotent.

## Logging out everywhere

To invalidate all sessions for the current user (e.g. after a credential leak), use the portal's **Settings → Sessions → Revoke all** action. There is no API endpoint for this today.
