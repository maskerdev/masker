**BLUF:** This example walks an end‑to‑end patient intake voice agent built on Vapi + Masker. The agent collects name, DOB, insurance, chief complaint, and medications in a 3–5 minute call. Every PHI field is tokenized in transit; OpenAI's logs only see Masker tokens; the per‑session audit report has the full BAA chain.

Source repo (illustrative): [github.com/masker-dev/example-patient-intake](https://github.com/masker-dev/example-patient-intake) — _placeholder; ships May 30, 2026 with the production beta._

## Architecture

```
Caller ↔ Vapi ──► Masker proxy ──► OpenAI (gpt-4o-mini) ──► Masker ──► Vapi ↔ Caller
                  │                                          │
                  detect + tokenize                          rehydrate
                  (regex + Gemma-4 NER)                      from per-session vault
                                                             │
                                                             └► Function call: createPatientRecord(...)
                                                                Masker rehydrates the args before they hit your EHR.
```

## Prompt skeleton

```yaml
# Vapi assistant config (excerpt)
firstMessage: "Hi, this is the Westside Family Medicine intake line. To get started, can I get your full name?"

systemPrompt: |
  You are a friendly patient intake coordinator for Westside Family Medicine.
  Collect: full name, DOB, insurance carrier and member ID, chief complaint,
  current medications, and pharmacy preference. Ask one or two questions at a
  time. Confirm sensitive fields (DOB, insurance) by reading them back.
  When complete, call the createPatientRecord function.

model:
  provider: custom-llm
  url: https://masker-voice.fly.dev/proxy/{agent_id}/v1/chat/completions
  model: gpt-4o-mini
  temperature: 0.3
```

## Function call schema

```json
{
  "name": "createPatientRecord",
  "description": "Create a patient intake record after the conversation is complete.",
  "parameters": {
    "type": "object",
    "properties": {
      "full_name": { "type": "string" },
      "dob": { "type": "string", "format": "date" },
      "insurance_carrier": { "type": "string" },
      "insurance_member_id": { "type": "string" },
      "chief_complaint": { "type": "string" },
      "current_medications": { "type": "array", "items": { "type": "string" } },
      "pharmacy_name": { "type": "string" }
    },
    "required": ["full_name", "dob", "insurance_carrier", "insurance_member_id", "chief_complaint"]
  }
}
```

Masker tokenizes the PHI fields (`full_name`, `dob`, `insurance_member_id`) in the request to OpenAI, and rehydrates them in the function call arguments returned to Vapi — so when Vapi POSTs to your `createPatientRecord` webhook, it has the original values.
